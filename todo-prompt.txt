>**Architecture**
>The app is a single HTML file: `doable.html`. It contains all HTML structure, CSS (Nord dark/light theme), and JavaScript (app logic, data persistence, rendering). No frameworks, no build tools, no network requests. All data persists in `localStorage` with the `doable_` prefix.
>
>**Key design decisions**
>- Single file only — CSS and JS are embedded via `<style>` and `<script>` tags
>- No hash-based routing needed — JS state variable `currentPage` switches views
>- All data is synchronous `localStorage` reads/writes
>- Activity log auto-truncates at 500 entries
>- Soft-delete: tasks get a `deleted_at` timestamp, hidden from main views
>- Recurring tasks (daily/weekly/monthly): on marking done, JS creates next instance
>- Focus goals stored per-date in `doable_focus`, auto-clears old dates
>- Backups are JSON file downloads containing all localStorage data
>- Export formats: JSON, CSV, Markdown table (generated in JS, downloaded)
>- Theme toggle switches `data-theme` attribute on `<html>`, CSS variables do the rest
>- UI follows Tufte principles: no box borders, no shadows, no gradients, color only for semantic encoding, sparklines instead of charts, labels on data not legends
>
>**Features**
>- 7 views: Dashboard (scratch notes + status + analytics + activity), Tasks (list/kanban/calendar), Focus (goals + timer + confetti), Bin, Activity Log, Settings
>- Dashboard: scratch notes (CRUD + pin), focus goals, motivational quote, overdue/due today lists, count stats, bar charts by priority/category, weekly/monthly recaps, 30-day sparkline, recent activity
>- Tasks: 3 view modes (list with sort/filter/group/paginate, kanban with drag-and-drop, calendar with month grid and expandable days), quick-add inline, global search
>- Task detail modal: inline editing for all fields, tag autocomplete, dependencies, annotations timeline, recurrence, soft-delete
>- Focus: select 1-3 goals, cycle status with click, progress counter, confetti on all done, auto-resets daily
>- Bin: list deleted tasks, restore individual, empty with confirmation
>- Activity log: chronological feed filterable by action type, keeps 500 entries
>- Settings: theme, date mode (smart/ISO), pagination size, backup download/restore upload, export (JSON/CSV/MD), clear all data
>- Sample data on first run (3 tasks + 1 scratch note)
