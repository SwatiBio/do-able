# Do-able - Data API Reference

Do-able uses a **dual-layer data architecture**: the frontend caches all data in `localStorage` for instant synchronous reads, while background HTTP calls sync mutations to a FastAPI backend that persists to SQLite.

## Conventions

- `localStorage` keys are prefixed with `doable_`
- Dates: ISO 8601 (`2026-06-25`) for due dates and start dates, ISO datetime for timestamps
- All IDs are unique strings generated by `Date.now().toString(36) + Math.random().toString(36).slice(2,6)`
- Soft-delete: tasks with `deleted_at` set are hidden from main views, visible in Bin

## Data Flow

```
User Action → Event Handler → JS Function → localStorage (sync, instant)
                                                  ↓
                                          Background HTTP → FastAPI → SQLite
```

- All reads are synchronous `localStorage` operations (zero latency)
- All writes update `localStorage` immediately, then fire a debounced API call
- `_initApi()` loads fresh data from the API on startup and merges into `localStorage`
- `_syncFull()` debounces bulk sync (800ms) posting all tasks/notes/config to `POST /api/sync/full`
- If the server is offline, `localStorage` still works — data syncs when connectivity returns

## Frontend Storage Keys

### `doable_tasks`

Array of task objects.

```json
[
  {
    "id": "k3x8a9b2c",
    "title": "Buy milk",
    "description": "",
    "status": "not_started",
    "priority": "high",
    "due_date": "2026-06-25",
    "start_date": null,
    "time": null,
    "category": "personal",
    "tags": ["errands"],
    "recur": null,
    "depends_on": [],
    "notes": [],
    "created_at": "2026-06-21T10:00:00.000Z",
    "updated_at": "2026-06-21T10:00:00.000Z",
    "deleted_at": null,
    "_sample": true
  }
]
```

- `status`: `"not_started"` | `"started"` | `"done"`
- `priority`: `"high"` | `"medium"` | `"low"`
- `due_date`: `"YYYY-MM-DD"` string or `null`/empty
- `start_date`: `"YYYY-MM-DD"` string or `null`/empty
- `time`: `"HH:mm"` string or `null`/empty
- `recur`: `"daily"` | `"weekly"` | `"monthly"` | `null`
- `depends_on`: array of task IDs that must be done first
- `notes`: array of `{ id, text, timestamp }` annotations
- `_sample`: optional flag set on sample tasks; used by "Remove Samples" in Settings

### `doable_activity`

Chronological array of activity entries (newest first). Keeps last 500, capped to last 10 days.

```json
[
  {
    "id": "m4x9c2d1e",
    "task_id": "k3x8a9b2c",
    "action": "created",
    "details": "Buy milk",
    "timestamp": "2026-06-21T10:00:00.000Z"
  }
]
```

Actions: `created`, `completed`, `deleted`, `restored`, `updated`

### `doable_config`

```json
{
  "theme": "nord-dark",
  "date_mode": "smart",
  "per_page": 25,
  "frog_enabled": false
}
```

- `theme`: `"nord-dark"` | `"nord-light"` | `"system"`
- `date_mode`: `"smart"` (relative: "today", "in 3 days") | `"iso"`
- `per_page`: integer, pagination size
- `frog_enabled`: boolean, show/hide frog companion

### `doable_notes`

Array of scratch pad notes from the Dashboard.

```json
[
  {
    "id": "n1a2b3c4d",
    "text": "Ideas for project...",
    "pinned": true,
    "created_at": "2026-06-21T10:00:00.000Z",
    "updated_at": "2026-06-21T10:00:00.000Z"
  }
]
```

### `doable_focus`

Map of date → task ID array. Auto-clears non-today dates on init. Max 3 goals per day.

```json
{
  "2026-06-21": ["k3x8a9b2c", "m7d4e5f6g"]
}
```

### `doable_taskColumns`

Array of visible column keys in task list view. Persisted across sessions.

```json
["priority", "status", "due", "tags", "category", "time", "annotations", "dependencies", "recur"]
```

Optional columns (hidden by default): `time`, `tags`, `category`, `annotations`, `dependencies`, `recur`.

## Frontend Functions (JS API)

All functions are defined in `doable.html` and globally accessible.

| Function | Description |
|----------|-------------|
| `getTasks()` | Returns array of all tasks (from localStorage cache) |
| `saveTasks(tasks)` | Replaces stored tasks array + fires API sync |
| `getActivity()` | Returns activity log |
| `saveActivity(entries)` | Replaces stored activity |
| `getConfig()` | Returns config object |
| `saveConfig(config)` | Saves config + fires API sync |
| `getNotes()` | Returns scratch notes |
| `saveNotes(notes)` | Saves scratch notes + fires API sync |
| `getFocus()` | Returns focus goals map |
| `saveFocus(focus)` | Saves focus goals map + fires API sync |
| `logActivity(taskId, action, details)` | Adds entry to activity log, auto-truncates to 500 |
| `showTaskDetail(id)` | Opens task detail full-page editor |
| `handleRecurrence(task)` | Creates next instance of a recurring task on completion |
| `createBackup()` | Downloads all data as JSON file |
| `restoreBackup(event)` | Uploads and restores data from JSON backup file |
| `exportData(format)` | Exports tasks as JSON, CSV, or Markdown |
| `quickAddTask()` | Creates a new task with just a title |
| `toggleTaskDone(id, checked)` | Toggles task completion, fires confetti, handles recurrence |
| `loadSampleData(quiet)` | Adds 20 sample tasks + logs them |
| `removeSampleTasks()` | Removes all tasks with `_sample` flag |
| `toggleFrog(on)` | Enables/disables the frog companion |
| `fireConfetti()` | Triggers confetti animation |
| `toast(msg, type)` | Shows a toast notification (success/error/info) |
| `navigateTo(page)` | Switches between app pages with unsaved-changes check |
| `cycleTheme()` | Cycles through Nord Dark → Nord Light → System |
| `pickRandomTask()` | Picks a random incomplete task for the roulette widget |

### API Client Functions (internal)

| Function | Description |
|----------|-------------|
| `_initApi()` | Loads tasks, config, notes, focus from API on startup |
| `_syncFull()` | Debounced (800ms) bulk sync to `POST /api/sync/full` |
| `_syncCfg()` | PUT config to `/api/config` |
| `syncFocus()` | PUT focus to `/api/focus` |

## Backend REST API

Base URL: `http://localhost:8000`

### Tasks

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/tasks` | List all tasks (supports query params: search, status, priority, category, due) |
| `POST` | `/api/tasks` | Create a new task |
| `GET` | `/api/tasks/{task_id}` | Get a single task |
| `PUT` | `/api/tasks/{task_id}` | Update a task |
| `DELETE` | `/api/tasks/{task_id}` | Soft-delete a task |
| `POST` | `/api/tasks/{task_id}/done` | Mark task done (triggers recurrence if recurring) |
| `POST` | `/api/tasks/{task_id}/undone` | Revert a done task to `not_started` |
| `POST` | `/api/tasks/{task_id}/note` | Add an annotation to a task |

### Config

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/config` | Get config |
| `PUT` | `/api/config` | Update config |

### Focus Goals

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/focus` | Get focus goals map |
| `PUT` | `/api/focus` | Update focus goals map |

### Scratch Notes

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/notes` | List all scratch notes |
| `POST` | `/api/notes` | Create a scratch note |
| `PUT` | `/api/notes/{note_id}` | Update a scratch note |
| `DELETE` | `/api/notes/{note_id}` | Delete a scratch note |

### Sync

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/sync/full` | Bulk import tasks, notes, and config. Body: `{ tasks: [...], notes: [...], config: {...} }`. Returns `{ id_map: { "old_id": "new_id" }, tasks: [...], notes: [...] }` for frontend ID reconciliation. |

### Search

### Backups

### Activity Log

## Features

| Feature | Implementation |
|---------|---------------|
| **Recurring** | On marking done, `handleRecurrence()` creates next instance with incremented due date |
| **Soft-delete** | `deleted_at` timestamp set, hidden from main views, visible in Bin |
| **Annotations** | Notes array on task object, shown in task detail page with timestamps |
| **Dependencies** | `depends_on` array of task IDs, with incompletion warnings and "blocks N tasks" indicator |
| **Search** | Client-side `String.includes()` across title + description, debounced 300ms |
| **Relative dates** | `fmtDate()` converts ISO to "today", "in 3 days", "yesterday", etc. |
| **Export** | `exportData()` generates JSON/CSV/MD and triggers download |
| **Backup/restore** | JSON file download/upload of all localStorage data (tasks, activity, config, notes, focus) |
| **Focus goals** | Per-date map in `doable_focus`, max 3 per day, auto-clears daily on init |
| **Overdue detection** | Client-side comparison of `due_date` against `todayStr()` |
| **Activity log** | Array in localStorage, keeps last 500 / last 10 days |
| **Theme** | CSS custom properties, `data-theme` attribute toggle, dark/light/system with `prefers-color-scheme` listener |
| **Frog companion** | SVG frog with 7 states, random auto-cycle, click-to-hop, modal peek, toast rider, confetti trigger |
| **Sort popover** | Two-step popover: select field → select direction; chips shown in sort bar |
| **Data filters** | Filter by status, priority, category, due date range (today/overdue/week/month/none) |
| **Column visibility** | Toggle columns via checkbox menu, persisted to `doable_taskColumns` |
| **Sample data** | Tasks with `_sample` flag; 20 tasks + 4 scratch notes loaded on first run, removable via Settings |
| **Calendar** | Month grid + week view; multi-day bars, recurring previews, drag-drop reschedule, jump-to-date, today pulse animation, overdue glow |
| **Kanban** | Grouped by category with drag-and-drop; shows task count per column; drop zone |
| **Eisenhower Matrix** | 2×2 grid (Do First/Schedule/Delegate/Eliminate) based on priority × due date urgency |
| **Heatmap** | GitHub-style 53-week contribution grid showing task completion density |
| **Task roulette** | Random incomplete task picker on dashboard |
| **Category management** | Settings section to delete unused categories; tasks become uncategorized |
| **Unsaved changes** | `detailDirty` flag triggers `confirm()` dialog when navigating away from task detail |
| **Confetti** | Falls from top with random colors/shapes; triggered on task done and all focus goals complete |
| **Background sync** | All writes update localStorage instantly + debounced API call to persist to SQLite |
