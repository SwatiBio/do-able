## State Machine

Valid Task status transitions:

```
not_started ‚îÄ‚îÄ‚Üí in_progress
in_progress  ‚îÄ‚îÄ‚Üí done
in_progress  ‚îÄ‚îÄ‚Üí cancelled
done         ‚îÄ‚îÄ‚Üí in_progress   (reopen ‚Äî back to active)
done         ‚îÄ‚îÄ‚Üí deleted       (soft-delete, moves to bin)
cancelled    ‚îÄ‚îÄ‚Üí not_started   (reopen ‚Äî fresh start)
cancelled    ‚îÄ‚îÄ‚Üí deleted       (soft-delete, moves to bin)
deleted      ‚îÄ‚îÄ‚Üí not_started   (restore from bin)
```

### Rules

- `not_started` ‚Üí `in_progress` is the only forward path
- `done` reopens to `in_progress` (keeps active state)
- `cancelled` reopens to `not_started` (truly fresh start)
- `done`/`cancelled` ‚Üí `deleted` moves to bin (keeps main views uncluttered)
- `deleted` can always restore to `not_started`

## Architecture Decisions

### File Split

Split the monolithic `doable.html` into `index.html` + `src/`:

| File | Responsibility |
|------|---------------|
| `index.html` | HTML shell, CSS, PWA manifest link |
| `src/app.js` | Data layer (localStorage, API client, save/load) |
| `src/ui.js` | Rendering (all render* functions, modals, navigation) |
| `src/events.js` | Event handlers (toggleTaskDone, cycleDashFocus, etc.) |

No build step. `<script>` tags loaded in order.

### Offline Write Queue

Instead of silently dropping failed API calls:
- Queue each individual mutation (edit task X, add task Y, delete task Z) with a sequence number
- Persist queue in localStorage
- Replay in order when connection returns
- Use `navigator.onLine` + `window.online` event listener to trigger replay

Per-mutation queue (option B) chosen over full-sync replay (option A) to avoid conflicts when editing while offline.
